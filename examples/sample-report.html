这是一个Claude Code的技能（skill），专门帮不懂设计的人做出漂亮的网页版演示文稿。 它的核心理念是"用嘴做PPT"——你只需要描述你想要什么，它就会生成几种视觉风格让你选，选完直接给你一个可以发链接的网页幻灯片。

最大的特点是零依赖——输出的就是一个HTML文件，没有npm包、没有框架、10年后还能打开。

⚙️ 功能实现详解

✨ 从零创建演示文稿

💡 原理：用户描述需求 → AI理解内容 → AI生成风格选项 → 用户选择 → AI生成完整HTML

🔄 代码流程：

用户输入需求 → [理解内容意图] → [生成3种风格预览] →
[用户选择] → [根据风格生成HTML] → [浏览器打开]

🔑 关键技术点：

CSS变量主题 渐进式披露 Viewport Fitting

📖 详细解释：

AI首先理解用户想要什么样的演示（内容是什么、传达什么情绪）， 然后生成3种不同风格的预览让用户选择，而不是问"你喜欢什么颜色"这种抽象问题。 用户选好后，AI根据选择的风格预设，生成完整的HTML文件。

📄 PPT转换功能

💡 原理：用python-pptx库提取PPT中的文字、图片 → AI理解内容结构 → 映射到网页幻灯片

🔄 代码流程：

上传.pptx → [extract-pptx.py提取] → [AI分析内容结构] →
[生成HTML版式] → [保留原版素材] → [选择风格]

🔑 关键技术点：

python-pptx库 XML解析 内容映射

📖 详细解释：

extract-pptx.py 是一个Python脚本，它解压PPT文件（.pptx就是zip包）， 解析其中的XML结构，提取每张幻灯片的文字、图片、布局信息。 然后这些信息传给AI，AI再生成对应的网页版本。

🔄 数据流向

用户创建演示的整体流程：

用户描述需求 → AI理解内容 → 生成风格预览 → 用户选择 → 生成HTML

步骤1：内容收集 — AI问用户要放什么内容、传达什么情绪

步骤2：风格生成 — AI根据内容生成3种风格预览（带实际颜色和排版）

步骤3：用户选择 — 用户说"第二个"，不需要解释为什么

步骤4：HTML生成 — AI根据选择的风格，生成完整的单HTML文件

步骤5：本地预览/发布 — 浏览器打开，或一键部署到Vercel
